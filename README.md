# s3seeding — засев S3-хранилища

Утилита для непрерывного наполнения S3-совместимого хранилища случайными
объектами. Заливает файлы заданной раскладки по размерам до остановки (`Ctrl-C`)
или достижения лимита

Ключ объекта строится из sha512 его содержимого по продакшн-схеме:
`<символы 1-4 sha512>/<символы 5-8>/<полный sha512>` (например `abcd/ef01/abcdef01…`).

## Требования

- Go 1.26+ (см. `go.mod`).
- Сетевой доступ к хранилищу и **уже существующий бакет**

## Быстрый старт

```bash
cp config.example.json config.json   # заполнить endpoint/bucket/ключи хранилища
make build                           # собрать бинарник ./s3seeding
make seed                            # засеять хранилище (STORAGE=zakroma_01)
```

Секреты можно не хранить в конфиге, а передавать через переменные окружения.
Имя переменной: `S3BENCH_<NAME>_<FIELD>`, где `<NAME>` — имя хранилища в верхнем
регистре (не-буквенно-цифровые символы → `_`):

```bash
export S3BENCH_ZAKROMA_01_ACCESS_KEY=...
export S3BENCH_ZAKROMA_01_SECRET_KEY=...
```

Поддерживаются `*_ACCESS_KEY`, `*_SECRET_KEY`, `*_ENDPOINT`, `*_BUCKET` —
значение из окружения переопределяет конфиг.

## Конфигурация

См. `config.example.json`. Поля:

- `part_size_mb` — размер части multipart-загрузки (≥ 5).
- `storages[]` — список хранилищ:
  - `name` — имя (используется в `-storage` и в именах env-переменных);
  - `endpoint` — URL со схемой (`https://host`) или `host:port`;
  - `bucket` — существующий бакет;
  - `access_key` / `secret_key` — ключи (можно через окружение);
  - `path_style` — path-style адресация бакета (`true` для MinIO-совместимых);
  - `insecure_skip_verify` — не проверять TLS-сертификат.
- `seed` — раскладка засева:
  - `small_pct` / `medium_pct` / `large_pct` — доли классов в процентах
    (в сумме > 0, обычно ~100);
  - `small` / `medium` / `large` — диапазоны размеров `{ "min_kb", "max_kb" }`.

  Если секция `seed` опущена, по умолчанию: 100% мелких объектов (1КБ–100КБ).

Пример раскладки из `config.example.json`: 98% мелких (1КБ–100КБ),
1.8% средних (100КБ–10МБ), 0.2% больших (10МБ–500МБ).

## Команда seed

```bash
# засеять хранилище zakroma (имя из config.json), 8 воркеров, до прерывания:
./s3seeding seed -config config.json -storage zakroma -workers 8

# опциональные лимиты (0 = без лимита):
./s3seeding seed -storage zakroma -max-objects 1000000      # стоп после N объектов
./s3seeding seed -storage zakroma -max-bytes 536870912000   # стоп после ~500 ГБ
```

Флаги:

| Флаг           | По умолчанию  | Назначение                                  |
| -------------- | ------------- | ------------------------------------------- |
| `-config`      | `config.json` | путь к JSON-конфигу                          |
| `-storage`     | `zakroma_01`  | имя хранилища из конфига                     |
| `-workers`     | `8`           | число параллельных загрузчиков              |
| `-max-objects` | `0`           | остановиться после N объектов (0 = без лимита) |
| `-max-bytes`   | `0`           | остановиться после N байт (0 = без лимита)  |

Каждые 5 секунд печатается прогресс (число объектов по классам, объём, об/с,
MB/с, ошибки), по завершении — итог. Если за первые 20 попыток не прошла ни одна
загрузка, засев останавливается (проверьте доступ/бакет).

## Детали ошибок

Ошибки операций (`put` / `bucket-exists` и др.) дополняются информацией из ответа
S3: HTTP-код, S3-код ошибки (`AccessDenied`, `NoSuchBucket`, …), сообщение
сервера, `request-id` и `host-id`. Если HTTP-ответа нет (DNS/TCP/TLS/таймаут),
пишется пометка о возможном сетевом сбое, HTTP-код = 0. Штатная отмена по
`Ctrl-C` (`context canceled`) ошибкой не считается.

## Команды make

```bash
make build   # собрать бинарник ./s3seeding
make seed    # засев хранилища (переменные CONFIG=, STORAGE=)
make test    # юнит-тесты с -race
```
